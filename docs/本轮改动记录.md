# 本轮改动记录

日期：2026-05-16 起  
整理：2026-05-24

说明：长期约定、反复会影响后续开发的坑，已拆到根目录 `AGENTS.md`。本文件只保留本轮变更摘要和待跟进项。

## 本轮主要调整

### 1. Agent 数据结构边界

- 强化 `space / position / item` 的模型语义边界。
- 历史误分类空间名、占位位置名会在 prompt/context 中标注，避免模型继续复用。
- `space_hint` 看起来像家具/位置时，作为 position 提示，而不是强制作为房间级空间。
- 后端继续保留用户确认优先：不静默改写模型返回的 `suggestion`。

涉及文件：

- `apps/api/src/agent.js`
- `apps/api/src/ai.js`
- `apps/api/src/locationRules.js`
- `apps/api/src/tools.js`
- `apps/api/src/store.js`

长期规则见：`AGENTS.md` 的 `Data Model Rules`。

### 2. 工具流程文案

- `save_items` 的流程文案从“待确认”改为“已整理成记录草稿”。
- 识别建议卡片主按钮从“确认”改为“保存”。

涉及文件：

- `apps/mobile/src/components/AgentWorkflow.js`
- `apps/mobile/src/components/SuggestionCard.js`

### 3. 空空间位置查询

- 修复 `list_positions` 在空间存在但没有位置时误报“没有找到空间”的问题。
- 现在区分“空间存在但位置为空”和“空间不存在”。

涉及文件：

- `apps/api/src/store.js`
- `apps/api/src/tools.js`

### 4. 多图上传交互

- 拍照/录像后保留草稿托盘，支持继续拍、追加相册、删除单张、统一发送。
- 相册选择在没有草稿时直接发送；如果输入框有说明文字，会随媒体一起发送。
- 已有草稿时，从相册补选只追加，不自动发送。
- 空间页和空间详情页会把 `camera / library` 来源传给助手页。
- 上传完成后，当前会话内图片保留本地预览，避免服务端缩略图回来时闪动。

涉及文件：

- `apps/mobile/src/screens/AssistantScreen.js`
- `apps/mobile/src/screens/SpacesScreen.js`
- `apps/mobile/src/screens/SpaceDetailScreen.js`
- `apps/mobile/src/sse.js`
- `apps/api/src/server.js`
- `apps/api/src/agent.js`

长期规则见：`AGENTS.md` 的 `Media Upload UX`。

### 5. Paywall 与顶部安全区

- Paywall 是绝对定位浮层，不能直接依赖主页面正常布局流的 safe area。
- 当前修法是在 Paywall 根节点使用稳定 top/bottom inset，避免标题首帧顶到状态栏或随后下跳。
- 主页面顶部栏额外上边距收紧，Paywall header 高度同步压缩，让首页、我的家、助手页和“我的”页顶部节奏更接近。

涉及文件：

- `apps/mobile/App.js`
- `apps/mobile/src/screens/PaywallScreen.js`

长期规则见：`AGENTS.md` 的 `Overlay And Safe Area Pitfall`。

### 6. 付费页展示

- 当前版本不再向用户展示“剩余 X 次识别”。
- 内部额度逻辑仍保留，用于识别前判断和付费状态刷新。

涉及文件：

- `apps/mobile/src/screens/PaywallScreen.js`
