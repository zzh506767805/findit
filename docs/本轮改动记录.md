### 7. 本地 iOS 模拟器启动记录

- 本机已安装并切换到 `Xcode 26.3 (17C529)`：`/Applications/Xcode.app/Contents/Developer`。
- 当前可用模拟器包含 `iPhone 17 Pro`，`simctl` 显示 iOS runtime 为 `iOS 26.3`。
- 本地 API 启动命令：

```bash
npm run dev:api
```

- 本地移动端 iOS 模拟器启动命令：

```bash
HTTP_PROXY=http://127.0.0.1:7897 HTTPS_PROXY=http://127.0.0.1:7897 npm --workspace apps/mobile run ios
```

- 首次启动 Expo iOS 时会下载 Expo Go。直连 GitHub release 可能很慢；本机 Clash Verge 系统代理端口为 `127.0.0.1:7897`，显式设置 `HTTP_PROXY` / `HTTPS_PROXY` 后下载正常。
- 如 Expo CLI 下载 Expo Go 长时间卡住，可手动下载并安装模拟器版本：

```bash
curl -x http://127.0.0.1:7897 -L \
  -o /tmp/Expo-Go-54.0.7.tar.gz \
  https://github.com/expo/expo-go-releases/releases/download/Expo-Go-54.0.7/Expo-Go-54.0.7.tar.gz
mkdir -p "$HOME/.expo/ios-simulator-app-cache/Expo-Go-54.0.7.tar.app"
tar -xzf /tmp/Expo-Go-54.0.7.tar.gz \
  -C "$HOME/.expo/ios-simulator-app-cache/Expo-Go-54.0.7.tar.app"
xcrun simctl install booted "$HOME/.expo/ios-simulator-app-cache/Expo-Go-54.0.7.tar.app"
```

- 2026-05-24 已验证：API、Metro、Expo Go、`iPhone 17 Pro` 模拟器均可启动；模拟器进入 FindIt 登录页。

### 8. 会员状态即时刷新

- 2026-06-28 排查充值后会员状态不立即刷新的原因：旧流程在通知 App 全局状态刷新前，先等待 App Store 的结束交易步骤；如果该步骤卡住，页面需要重启后才会从服务器重新读到会员状态。
- 当前移动端流程改为：后端购买验证成功后，先把返回的会员状态写回 App 全局状态并关闭付费页，再异步执行 App Store 交易收尾。
- App 全局状态在收到购买结果后，还会后台再拉一次 `/user/credits`，避免只依赖购买接口返回值。
- 本地 API 在非生产环境支持移动端开发模式的模拟开通，方便用 iOS 模拟器验证会员状态刷新。
- Expo 补丁版本对齐到 `54.0.35`，让发布前检查恢复通过。
- TestFlight 提交时发现 App Store Connect 已有 build number `4`，本轮将 iOS build number 递增到 `5` 后重新打包提交。
- 真机 TestFlight 复测发现：支付完成后 Apple 回调没有稳定触发，点“恢复购买”才能刷新。移动端购买流程增加兜底：`purchaseItemAsync` 返回后主动查询购买历史并验证 receipt，等价于自动做一次恢复刷新；iOS build number 递增到 `6`。
