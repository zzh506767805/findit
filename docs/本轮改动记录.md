### 8. 会员状态即时刷新

- 2026-06-28 排查充值后会员状态不立即刷新的原因：旧流程在通知 App 全局状态刷新前，先等待 App Store 的结束交易步骤；如果该步骤卡住，页面需要重启后才会从服务器重新读到会员状态。
- 当前移动端流程改为：后端购买验证成功后，先把返回的会员状态写回 App 全局状态并关闭付费页，再异步执行 App Store 交易收尾。
- App 全局状态在收到购买结果后，还会后台再拉一次 `/user/credits`，避免只依赖购买接口返回值。
- 本地 API 在非生产环境支持移动端开发模式的模拟开通，方便用 iOS 模拟器验证会员状态刷新。
- Expo 补丁版本对齐到 `54.0.35`，让发布前检查恢复通过。
- TestFlight 提交时发现 App Store Connect 已有 build number `4`，本轮将 iOS build number 递增到 `5` 后重新打包提交。
- 真机 TestFlight 复测发现：支付完成后 Apple 回调没有稳定触发，点“恢复购买”才能刷新。移动端购买流程增加兜底：`purchaseItemAsync` 返回后主动查询购买历史并验证 receipt，等价于自动做一次恢复刷新；iOS build number 递增到 `6`。

### 11. 代码 review 清理 + 退出登录（2026-07-04）

- 全量 review 后端与前端核心代码，发现的主要问题（暂未修，见下方待跟进）：
  - `/media/:id/thumb|original` 无鉴权，拿到 media_asset_id 即可换取 1 小时 SAS URL（IDOR）。
  - 注销账号只删数据库记录，Blob Storage 里的照片/视频没删，与隐私政策承诺不符。
  - `/agent/query` 查询无频控、无长度限制、空 query 也会调 AI，成本不设防。
  - App 启动恢复会话时，`/user/credits` 网络错误也会清 session 把用户登出（应只在 401 时清）。
  - 其他中低优先级：IAP receipt 未校验 bundle_id、`addSasToUrls` 全量正则替换有 JSON 转义隐患、`searchItems` LIKE 通配符未转义、deleteSpace/deletePosition/deleteItem 无事务、`ON CONFLICT (apple_user_id)` 与 partial index 的可移植性问题。
- 删除死代码：整个 `apps/api/src/ai.js`（被 agent.js 取代的旧版识别模块）；store.js 中未使用的 `findOrCreateItem` / `createItemRecord` / `updateMediaAssetPosition` / `formatLocationPath` / `updateMessageSuggestion`；server.js 中对应的无用 import。
- 新增退出登录：付费页“账号与数据”区加“退出登录”按钮（确认弹窗，说明数据不删除），App.js 抽出 `resetToLogin()` 供退出登录和删除账号共用（清 session + 重置全部状态回登录页）。
- 修正文档：免费额度实际为 3 次（代码 `DEFAULT_FREE_CREDITS = 3`）+ 新人 15 天/30 次礼包 + 邀请码双方各 15 次，CLAUDE.md 原写“免费 10 次”已过时。
- 新增每日 AI 配额（纯后端）：新建 `usage_daily` 表（user_id + day 主键，query_count / analyze_count），`consumeDailyAiQuota()` 原子自增并按会员状态判限——免费用户对话和识别各 1 次/天（产品决策，收紧转化）；会员各 50 次/天（防滥用兜底），超限 429。领了新人礼包的用户 15 天内按会员档计。`/agent/query` 同时校验空内容（400）和 500 字上限（400）。已本地验证：空内容/超长返回 400，超限触发 429，注销账号会连带清理 usage_daily。
- 配额语义与检查顺序（采纳外部 review 意见）：
  - `/agent/analyze` 的检查提前到解析/上传文件之前：先只读预检余额（不足直接 403，不计配额），再计每日配额（超限 429），避免超限/无额度用户白白消耗上传和存储成本；正式的 `consumeCredit` 原子扣减仍在上传成功后执行。
  - 配额计"尝试次数"，但 AI 调用失败会回滚：analyze 失败时 `refundCredit` + `refundDailyAiQuota` 一起退；query 失败也退每日配额。参数错误（如空文件）不回滚，垃圾请求也占限额。
  - "今天"按北京时间算（`now() AT TIME ZONE 'Asia/Shanghai'`），线上 DB 是 UTC，直接用 CURRENT_DATE 会导致早上 8 点才重置。
